Query Output Format

Fields of the query result:

  • Transcript_ID - RefSeq transcript ID;
  • Chrom - chromosome or locus name;
  • Beg - start genome position of the translation region for the peptide *;
  • End - end genome position of the translation region for the peptide *;
  • Upstream frame shifts - frame shifting variations (with allele frequency values) in upstream of the translation region which can induce the alternative frame translation; if several combinations of frame shifts can induce the current frame the full set of the combinations will be presented with ‘OR’ delimiter; the canonical translation frame is denoted by '-' symbol;
  • Variations inside - genome variations (with allele frequency values and relative nucleotide positions) inside the translation region; synonymous variations are enclosed in square brackets;
  • Peptide - peptide sequence;

* - if the translation region starts and/or ends on a genome insertion the position will be presented as the insertion position plus a half;

About the PeptoVarDB project

PeptoVarDB is an on-line database of annotated peptides based on human genome annotation and public data of human genome common variants created with PeptoVar in the pupulation mode. PeptoVar (Peptides of Variations) is a program for annotation of genomic variants in protein coding genes and generation of annotated peptides (open-projects.github.io/PeptoVar/). The main idea of PeptoVar is implementation of the potential compound effects of combining alternate alleles across multiple variant loci in sequence translation. PeptoVar is helpful for in-silico prediction of new minor histocompatibility antigens (MiHa), simulation of stem sells transplantation or as a peptide database generator for shotgun proteomic studies of immunopeptidome.
The current release of PeptoVarDB based on RefSeq Annotation release 108 (www.ncbi.nlm.nih.gov/refseq/) and dbSNP Build 151 (www.ncbi.nlm.nih.gov/projects/SNP/). Only genome variants with Allele Frequency (AF) > 0.01 was used as the input data for the PeptoVar program. The program was started with the next parameters:
PeptoVar.py -tagaf CAF -minaf 0.01 -peptlen 8 9 10 11 12 -var nonsyn -indir ./input
